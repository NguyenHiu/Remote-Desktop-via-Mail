Remote Desktop via Mail
- Instruction -
Hello there,
This is a brief guide to the Remote-Desktop-via-Mail program. This mail will show you
how to send a syntactically correct request to control the remote computer in your own
way!
This instruction will go through 20 commands (each with an example if it requires a
content that cannot be left blank). If the command does not request the content, the
content will not be mentioned in this instruction.
The group of commands will be divided into two groups: commands about the control
feature and commands that are systematic .
1. Control commands:
a. PROCESS:
 View a list of running processes:
Subject: RDM-LIST PROCESS
 Kill a particular process by specifying the ID:
Subject: RDM-KILL PROCESS
Content: <ID of process that you want to kill>
Example: Content: 13579
b. APP:
 View a list of running applications:
Subject: RDM-LIST APP
 Kill a particular application by specifying the ID:
Subject: RDM-KILL APP
Content: <ID of application that you want to kill>
Example: Content: 24680
c. SCREEN:
 Shot screen:
Subject: RDM-CAPTURE SCREEN
 Record screen:
Subject: RDM-RECORD SCREEN
Content: <input recording time, unit: seconds>
Example: Content: 10
d. WEBCAM:
 Shot webcam:
Subject: RDM-SHOT WEBCAM
 Record webcam:
Subject: RDM-RECORD WEBCAM
Content: <input recording time, unit: seconds>
Example: Content: 10
2.Systematic commands:
a. FILE SYSTEM:
 View a tree of a particular system file:
Subject: RDM-VIEW FILE SYSTEM
Content: <path of the folder that you want to view>
Note:The content can be blank, that means "I want to view the root "
Example: Content: C:\Users\Admin
 Copy a file:
Subject: RDM-COPY FILE SYSTEM
Content: "<path of the file you want to copy> " "<path of the folder you want to copy to> "
Example: Content:
"C:\Users\Admin\Documents\GitHub\report.pdf " "C:\Users\Admin\Documents "
 Download a file:
Subject: RDM-DOWNLOAD FILE SYSTEM
Content: <path of the file that you want to download>
Example: Content: C:\Users\Admin\Documents\instruction.pdf
b. KEYLOGGER:
Subject: RDM-KEYLOGGER
Content: <input tracking time, unit: seconds>
Example: Content: 10
c. SHUTDOWN:
Subject: RDM-SHUTDOWN
d. RESTART:
Subject: RDM-RESTART
e. REGISTRY:
 Create a registry key:
Subject: RDM-CREATE REGISTRY
Content: <Hotkey>,<path of the key (folder) you want to create key in>,<name of the new key>
Example: Content:
HKEY_CURRENT_CONFIG,Software\Fonts,NewFont
 Set value for a key:
Subject: RDM-SET REGISTRY
Content: <Hotkey>,<path of the key you want to set value>,<name of the value you want to set>,
<datatype of the value>,<value>
Example: Content:
HKEY_CURRENT_USER,Microsoft\Windows\CurrentVersion\Explorer\StartupApproved\Run,
OneDriveSetup,REG_BINARY,04 00 00 0 00 00 00 00 00 00 00 000
 Get the value of a key:
Subject: RDM-GET REGISTRY
Content: <Hotkey>,<path of the key you want to set value>,<name of the value you want to get>
Example: Content:
HKEY_CURRENT_USER,Microsoft\Windows\CurrentVersion\Explorer\StartupApproved\Run,OneDriveSetup
 Write a new key, value in the system:
Subject: RDM-WRITE REGISTRY
Content: <Blocks of code according to structure of REG file>
Note:The content can be blank, but email must attach a REG file.

 Delete a key:
Subject: RDM-DELETE KEY REGISTRY
Content: <Hotkey>,<path to the key you want to delete>
Note:If you delete a key having some subkeys, they will be deleted too.
Example: Content:
HKEY_CURRENT_CONFIG,Software\Fonts\NewFont
 Delete value of a key:
Subject: RDM-DELETE VALUE REGISTRY
Content: <Hotkey>,<path to the key you want to delete its value>,<name of the value you want to delete>
Example: Content:
HKEY_CURRENT_USER,Microsoft\Windows\CurrentVersion\Explorer\StartupApproved\Run,OneDriveSetup
